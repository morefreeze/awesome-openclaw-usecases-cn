# 活动嘉宾确认

你正在举办一个活动——晚宴、婚礼、公司团建——你需要从嘉宾名单中确认出席情况。手动拨打20多个电话很繁琐：你要反复联系，忘记谁说了什么，还要记录饮食限制或额外陪同人员。发短信有时管用，但人们会忽略消息。真正的电话通话回复率要高得多。

这个用例让OpenClaw使用[SuperCall](https://clawhub.ai/xonder/supercall)插件拨打你名单上的每位嘉宾，确认他们是否出席，收集任何备注，并为你整理成摘要。

## 功能介绍

- 遍历嘉宾名单（姓名+电话号码）并拨打每位嘉宾
- AI以友好的身份介绍自己是你的活动协调员
- 与嘉宾确认活动日期、时间和地点
- 询问他们是否出席，并收集任何备注（饮食需求、额外陪同人员、到达时间等）
- 所有电话完成后，整理摘要：谁确认出席，谁拒绝，谁没接电话，以及任何备注

## 为什么选择SuperCall

这个用例专门与[SuperCall](https://clawhub.ai/xonder/supercall)插件配合使用——而不是内置的`voice_call`插件。关键区别在于：SuperCall是一个完全独立的语音代理。通话中的AI角色**只能访问你提供的上下文**（角色名称、目标和开场白）。它无法访问你的网关代理、你的文件、你的其他工具或任何其他内容。

这对嘉宾确认很重要，因为：

- **安全**：通话另一端的人无法通过对话操纵或访问你的代理。没有提示注入或数据泄露的风险。
- **更好的对话**：因为AI专注于单一任务（确认出席），它会保持主题，比通用语音代理更自然地处理通话。
- **适合批量处理**：你要给不同的人打很多电话。每次通话后重置的沙盒化角色正是你想要的——对话之间不会相互影响。

## 所需技能

- [SuperCall](https://clawhub.ai/xonder/supercall) — 通过`openclaw plugins install @xonder/supercall`安装
- 带有电话号码的Twilio账户（用于拨出电话）
- OpenAI API密钥（用于GPT-4o实时语音AI）
- ngrok（用于webhook隧道——免费 tier 可用）

有关完整配置说明，请参见[SuperCall README](https://github.com/xonder/supercall)。

## 设置方法

1. 按照[设置指南](https://github.com/xonder/supercall#configuration)安装和配置SuperCall。确保在OpenClaw配置中启用了hooks。

2. 准备你的嘉宾名单。你可以直接粘贴到聊天中，也可以保存在文件中：

```text
嘉宾名单 — 夏季烧烤，6月14日星期六，下午4点，橡树街23号

- Sarah Johnson: +15551234567
- Mike Chen: +15559876543
- Rachel Torres: +15555551234
- David Kim: +15558887777
```

3. 提示OpenClaw：

```text
我需要你为我的活动确认出席情况。以下是活动详情：

活动：夏季烧烤
日期：6月14日星期六下午4点
地点：橡树街23号

这是我的嘉宾名单：
<在此粘贴你的嘉宾名单>

对于每位嘉宾，使用supercall拨打他们。使用身份“Jamie，[你的名字]的活动协调员”。每次通话的目标是确认他们是否出席，并记录任何饮食限制、额外陪同人员或其他评论。

每次通话后，记录结果。所有通话完成后，给我一个完整的摘要：
- 谁确认出席
- 谁拒绝
- 谁没接电话
- 每位嘉宾的任何备注或特殊要求
```

4. OpenClaw将使用SuperCall逐个拨打每位嘉宾，然后整理结果。你可以随时通过询问状态更新来检查进度。

## 关键见解

- **从小规模测试开始**：先尝试给2-3位嘉宾打电话，确保角色和开场白听起来合适。在拨打完整名单之前，你可以调整语气。
- **注意通话时间**：不要安排太早或太晚的通话。你可以告诉OpenClaw只在特定时间段内打电话。
- **查看记录**：SuperCall将通话记录保存到`~/clawd/supercall-logs`。第一批通话后浏览一下，看看对话情况如何。
- **未接听处理**：如果有人没接电话，OpenClaw可以记录下来，你可以决定以后是否重试或发短信跟进。
- **真实电话通话需要花钱**：每次通话使用Twilio分钟数。在Twilio账户中设置适当的限制，尤其是对于大型嘉宾名单。

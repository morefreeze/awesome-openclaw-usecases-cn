# 个人知识库(RAG)

你整天阅读文章、推文和观看视频，但永远找不到上周看到的那个东西。书签堆积如山，变得毫无用处。

这个工作流从你保存的所有内容中构建一个可搜索的知识库：

• 将任何URL拖到Telegram或Slack中，它会自动摄取内容（文章、推文、YouTube文字记录、PDF）
• 对你保存的所有内容进行语义搜索：“我保存了哪些关于代理内存的内容？”返回带有来源的排名结果
• 馈送到其他工作流中——例如，视频创意流水线在构建研究卡片时会查询知识库以获取相关的已保存内容

## 所需技能

- [knowledge-base](https://clawhub.ai)技能（或使用嵌入构建自定义RAG）
- `web_fetch`（内置）
- 用于摄取的Telegram主题或Slack频道

## 设置方法

1. 从ClawdHub安装knowledge-base技能。
2. 创建一个名为“knowledge-base”的Telegram主题（或使用Slack频道）。
3. 提示OpenClaw：
```text
当我在“knowledge-base”主题中拖放URL时：
1. 获取内容（文章、推文、YouTube文字记录、PDF）
2. 将其与元数据（标题、URL、日期、类型）一起摄取到知识库中
3. 回复确认：摄取了什么以及块数

当我在这个主题中提问时：
1. 对知识库进行语义搜索
2. 返回带有来源和相关摘录的顶级结果
3. 如果没有匹配的结果，告诉我

此外：当其他工作流需要研究（例如，视频创意、会议准备）时，自动查询知识库以获取相关的已保存内容。
```

4. 通过拖放一些URL并提问来测试，例如“我有哪些关于LLM内存的内容？”
